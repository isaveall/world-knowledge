Next.js 是一个基于 React 的开源全栈 Web 开发框架，由 Vercel 公司主导开发，旨在简化现代 Web 应用的构建流程。它在 React 的基础上提供了服务端渲染（SSR）、静态站点生成（SSG）、文件系统路由、API 路由等开箱即用的功能，显著提升了性能与 SEO 能力。

---

## 核心特性

- 服务端渲染（SSR）与静态生成（SSG）  
  Next.js 支持两种主要的预渲染方式：
  - SSG：在构建时生成 HTML 页面，适用于内容不频繁变化的页面（如博客、产品列表），加载速度快，SEO 友好。
  - SSR：在每次请求时动态生成 HTML，适用于数据实时性要求高的页面（如用户仪表盘）。

- 文件系统路由  
  页面路由基于 `pages` 目录下的文件结构自动创建，无需额外配置。例如：
  - `pages/index.js` → `/`
  - `pages/about.js` → `/about`
  - `pages/posts/[id].js` → 支持动态路由（如 `/posts/1`）

- 数据获取方法  
  - `getStaticProps()`：用于 SSG，构建时获取数据。
  - `getServerSideProps()`：用于 SSR，每次请求时获取数据。
  - `getStaticPaths()`：配合动态路由，预定义可生成的路径。

- 内置 API 路由  
  在 `pages/api/` 目录下创建的文件会自动成为 API 接口，实现前后端一体化开发。

- 优化体验  
  - 自动代码分割
  - 热模块替换（HMR）
  - 图片优化组件（`next/image`）
  - 支持 CSS Modules、Sass、Styled Components 等多种样式方案

---

## 适用场景

- 静态网站：博客、文档站、营销页面  
- 动态网站：电商、新闻平台、社交应用  
- 后台管理系统、PWA（渐进式 Web 应用）

---

## 快速上手

1. 确保已安装 Node.js（推荐 18+ 版本）
2. 创建项目：
   ```bash
   npx create-next-app@latest my-app
   cd my-app
   npm run dev
   ```
3. 访问 `http://localhost:3000` 即可查看默认页面

官方文档：[Next.js 官方文档](https://nextjs.org/docs)

---

## 注意事项

- Next.js 14.1.1 及以上版本修复了 SSRF 漏洞（CVE-2024-34351），建议使用最新稳定版以保障安全 。
- 虽然 Next.js 简化了配置，但需理解 SSR/SSG 的适用场景，避免误用影响性能 。
